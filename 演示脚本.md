# 演示脚本

> 30 秒展示核心交互链路。可在微信或 API 直接调用。

---

## 链路 1: 完整新闻交互 (30s)

```
用户: 今日热点
 Bot: [总览卡 PNG] 回复 1-12 或标题查看详情

用户: 3
 Bot: [详情卡 PNG] 已切到这条新闻详情

用户: 分析一下
 Bot: **核心判断:** ... **为什么值得看:** ... **接下来观察什么:** ...

用户: 看后续
 Bot: 近 7 天后续追踪: ... (或 "暂无后续报道")

用户: 清空上下文
 Bot: 上下文已清空。
```

**展示点**: 意图路由 → 卡片渲染 → AI 分析 → 追踪检索 → 会话管理

---

## 链路 2: Tool Calling (20s)

```
用户: 找两条AI新闻分析一下

 Bot: [内部: toolLoop]
      Step 1: fetch_news("AI", 2) → 英伟达B300, Google Gemini
      Step 2: analyze_article("英伟达B300") → 核心判断...
      → 合成回复

用户: 那后面那个也分析一下
 Bot: [ToolLoop]
      Step 3: analyze_article("Google Gemini") → 核心判断...
      → 合成回复
```

**展示点**: LLM 自主编排多步工具调用，ReAct 循环

---

## 链路 3: 模糊口语 → 准确路由 (15s)

```
用户: 有啥新鲜事
 Bot: [总览卡 PNG] ← L2 embedding 识别为 news_overview

用户: 那个芯片的详细说说
 Bot: [详情卡] ← L2 识别为 detail_followup

用户: 早上好
 Bot: 早上好~ ← L2 识别为 casual_chat (不走新闻管道)
```

**展示点**: L2 Embedding 原型分类泛化能力

---

## 链路 4: 置信度反问 (10s)

```
用户: 那个啥
 Bot: 你是想看刚才某条新闻的详情，还是想查新的新闻？
     直接说标题或发序号。

用户: 第3条
 Bot: [详情卡 PNG]
```

**展示点**: 低置信度不硬猜，根据对话状态给引导

---

## 链路 5: 负向测试 (10s)

```
用户: qwerty
 Bot: 不太确定你想做什么。你可以说 '今日热点' 看新闻、
      '今天天气怎么样' 查天气，或者直接发问题。

用户: /help
 Bot: 可用方式:
      1. 发"今日热点"查看最新热点列表
      2. 发"最近有什么新闻"查看新闻源层的数据
      ...
```

**展示点**: 不能处理时不崩溃，给有意义的引导

---

## 答辩/面试时的讲述顺序

1. **开场 (5s)**: "这是一个具备 Tool Calling 和三层意图路由的新闻助手 Agent"
2. **链路 1 (30s)**: 完整交互 —— 展示核心价值
3. **链路 3 (15s)**: 解释三层路由 —— "关键词管确定命令, Embedding 管口语变体"
4. **链路 2 (20s)**: 解释 Tool Calling —— "LLM 自己拆任务调工具"
5. **设计亮点 (30s)**: Java 2D 卡片为什么不用 HTML→图片、L3 教学 L2 的闭环、手写 Tool 协议为什么不套 LangChain
6. **已知限制 (10s)**: Memory 尚浅、L3 待升级、单用户
